# 영업 활동 지원 시스템 프로젝트 결과서

이 프로젝트는 영업 사원의 효율적인 고객 관리와 활동 기록을 지원하기 위한 모바일 최적화 웹 애플리케이션입니다. n8n 워크플로우와의 긴밀한 연동을 통해 실시간 기업 정보와 활동 데이터를 관리하며, Apple 스타일의 정갈하고 프리미엄한 UI를 제공합니다.

---

## 1. 프로젝트 개요
- **목적**: 고객사 정보 조회, 활동 일정 관리, 영업 기록 실시간 동기화
- **사용자 경험**: 직관적인 캘린더 인터페이스와 통합 검색 기반의 데이터 입력
- **핵심 연동**: n8n 워크플로우 기반의 외부 API(DART, 네이버 뉴스 등) 및 DB 연동

---

## 2. 주요 기능 상세

### 2.1. 영업 활동 캘린더 (CalendarPage)
- **실시간 일정 동기화**: 월/일자 변경 시마다 n8n 웹훅(`type: findact`)을 호출하여 최신 데이터를 반영합니다.
- **안정적인 데이터 매핑**: 서버의 비정형 데이터(`DATE`, `CORP_NM`, `ACT_REPORT` 등)를 클라이언트의 `SalesActivity` 규격으로 자동 변환합니다.
- **활동 등록 제약**: 데이터 무결성을 위해 고객사 명은 직접 입력을 제한하고, **통합 검색**을 통해서만 선택할 수 있도록 구현되었습니다.
- **일정 상세 조회**: 등록된 일정을 클릭하면 상세 내용(활동 기록, 상태, 시간 등)을 팝업 모달로 즉시 확인할 수 있습니다.

### 2.2. 고객 정보 조회 및 AI 분석 (SearchPage)
- **기업 심층 정보**: n8n을 통해 DART(전자공시) 데이터와 네이버 뉴스 동향을 가져와 영업 전략 수립에 필요한 인사이트를 제공합니다.
- **유연한 데이터 렌더링**: 재무 지표가 JSON 객체 또는 문자열 등 다양한 형태로 올 경우를 대비한 **방어적 파싱 로직**이 적용되어 화면 멈춤(Blank Page) 현상을 원천 차단했습니다.
- **AI 리포트**: 최신 뉴스를 기반으로 한 AI 분석 리포트를 카드 레이아웃으로 가독성 있게 제공합니다.

### 2.3. 영업 활동 기록 관리 (RecordsPage)
- **월 단위 관리**: 상단 월 이동 헤더를 통해 특정 달의 기록만 선별하여 조회할 수 있습니다.
- **실시간/로컬 병합**: 로컬에 저장된 임시 데이터와 n8n 서버의 실시간 데이터를 통합하여 끊김 없는 사용자 최신 정보를 유지합니다.
- **유형별 필터링**: 미팅, 전화, 이메일 등 활동 유형에 따른 빠른 필터링 기능을 제공합니다.

---

## 3. 기술 아키텍처 및 구현 특징

### 3.1. API 통신 레이어 (`n8n.ts`)
- **멀티 타입 웹훅**: 단일 엔드포인트에서 `type` 파라미터(`customerSearch`, `custinfo`, `actibity`, `findact`)에 따라 검색, 상세조회, 저장, 리스트 조회를 모두 처리합니다.
- **오타 및 특이 요구사항 반영**: 서버 측 요구 사항에 맞춰 `activity` -> `actibity` 식별자 매핑을 완벽하게 구현했습니다.

### 3.2. 상태 관리 및 유틸리티
- **Storage Utility**: `localStorage`를 활용하여 새로고침 시에도 데이터 유실을 방지합니다.
- **타입 안정성**: 모든 데이터는 `SalesActivity`, `CustomerInfo` 등 엄격한 TypeScript 인터페이스를 기반으로 관리됩니다.

### 3.3. 디자인 시스템 (UI/UX)
- **Aesthetic**: `backdrop-blur`, 부드러운 그림자(`shadow-lg`), 프리미엄한 타이포그래피(Outfit)를 적용한 Apple 스타일 디자인입니다.
- **Micro-interactions**: 버튼 클릭 효과, 모달 슬라이드 애니메이션 등을 통해 사용자 참여감을 높였습니다.

---

## 4. 최종 결과물 주요 가치
- **데이터 일관성**: 검색 기반 고객 선택으로 데이터 중복 및 오입력 방지.
- **실시간성**: 캘린더와 기록 페이지가 항상 최신 서버 데이터를 유지.
- **견고함**: 다양한 API 응답 환경에 대응하는 예외 처리로 서비스 가용성 확보.
